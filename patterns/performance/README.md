# Performance Patterns

**Focus:** latency, throughput, and cost — the patterns that make AI systems fast and affordable enough for production.

Patterns in this category follow the [MAP pattern template](../_template/PATTERN_TEMPLATE.md).
See the [Roadmap](../../ROADMAP.md) for the full backlog and status.

> No patterns published yet. Want to write one? Open a **"New Pattern"** issue and see
> [CONTRIBUTING.md](../../CONTRIBUTING.md).

## Planned patterns

- Streaming
- Prompt Cache
- Response Cache
- Parallel / Fan-out Retrieval
- Speculative Execution
- Request Batching
- Model Cascade (cheap-first)
- Token-Budget Optimization

## Contributing to this category

1. Claim a pattern above (or propose a new one) via a **New Pattern** issue.
2. Copy the [template](../_template/PATTERN_TEMPLATE.md) to `performance/<pattern-slug>/README.md`.
3. Open a PR into `dev`.
