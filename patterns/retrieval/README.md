# Retrieval Patterns

**Focus:** getting the right context in front of the model — chunking, search, reranking, and caching strategies that determine what the model actually sees.

Patterns in this category follow the [MAP pattern template](../_template/PATTERN_TEMPLATE.md).
See the [Roadmap](../../ROADMAP.md) for the full backlog and status.

> No patterns published yet. Want to write one? Open a **"New Pattern"** issue and see
> [CONTRIBUTING.md](../../CONTRIBUTING.md).

## Planned patterns

- Chunking Strategies
- Parent-Child (Small-to-Big) Retrieval
- Hybrid Search (dense + sparse)
- Metadata Filtering
- Multi-Query Retrieval
- Contextual Compression
- Semantic Cache
- Reranking
- HyDE
- Sentence-Window Retrieval
- Self-Query Retrieval
- Hierarchical / Recursive Retrieval
- Graph RAG
- Contextual Retrieval
- Late Chunking

## Contributing to this category

1. Claim a pattern above (or propose a new one) via a **New Pattern** issue.
2. Copy the [template](../_template/PATTERN_TEMPLATE.md) to `retrieval/<pattern-slug>/README.md`.
3. Open a PR into `dev`.
