# Tool Calling Patterns

**Focus:** letting models act through tools safely and reliably — selection, validation, parallelism, and error handling for function calling.

Patterns in this category follow the [MAP pattern template](../_template/PATTERN_TEMPLATE.md).
See the [Roadmap](../../ROADMAP.md) for the full backlog and status.

> No patterns published yet. Want to write one? Open a **"New Pattern"** issue and see
> [CONTRIBUTING.md](../../CONTRIBUTING.md).

## Planned patterns

- Basic Tool Calling
- Parallel Tool Calls
- Tool Selection / Filtering
- Dynamic Tool Loading
- Structured Output / Function Calling
- Tool Result Validation
- Tool Error Handling
- Confirmation-Gated Tools

## Contributing to this category

1. Claim a pattern above (or propose a new one) via a **New Pattern** issue.
2. Copy the [template](../_template/PATTERN_TEMPLATE.md) to `tool-calling/<pattern-slug>/README.md`.
3. Open a PR into `dev`.
