# Security Patterns

**Focus:** defending AI systems against injection, data leakage, and abuse — the guardrails that make an AI system safe to expose.

Patterns in this category follow the [MAP pattern template](../_template/PATTERN_TEMPLATE.md).
See the [Roadmap](../../ROADMAP.md) for the full backlog and status.

> No patterns published yet. Want to write one? Open a **"New Pattern"** issue and see
> [CONTRIBUTING.md](../../CONTRIBUTING.md).

## Planned patterns

- Prompt Injection Defense
- Secret Isolation
- Tenant Isolation
- PII Redaction
- Sandboxing
- Output Guardrails / Filtering
- Input Validation
- Least-Privilege Tool Access
- Rate Limiting & Abuse Prevention
- Content Moderation

## Contributing to this category

1. Claim a pattern above (or propose a new one) via a **New Pattern** issue.
2. Copy the [template](../_template/PATTERN_TEMPLATE.md) to `security/<pattern-slug>/README.md`.
3. Open a PR into `dev`.
