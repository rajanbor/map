# Context Management Patterns

**Focus:** budgeting and shaping the context window — deciding what goes into the prompt, in what order, and how to keep it within limits.

Patterns in this category follow the [MAP pattern template](../_template/PATTERN_TEMPLATE.md).
See the [Roadmap](../../ROADMAP.md) for the full backlog and status.

> No patterns published yet. Want to write one? Open a **"New Pattern"** issue and see
> [CONTRIBUTING.md](../../CONTRIBUTING.md).

## Planned patterns

- Context Window Budgeting
- Prompt Compression
- Context Summarization
- Structured Prompting
- Few-Shot Example Selection
- Context Ordering (lost-in-the-middle)
- Message Pruning
- Dynamic Context Assembly

## Contributing to this category

1. Claim a pattern above (or propose a new one) via a **New Pattern** issue.
2. Copy the [template](../_template/PATTERN_TEMPLATE.md) to `context/<pattern-slug>/README.md`.
3. Open a PR into `dev`.
